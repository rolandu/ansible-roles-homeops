# ansible-roles-homeops

> [!WARNING]
> I am still new to do this. Use at your own risk!

Ansible roles for home/family infrastructure (servers, laptops, workstations, etc.).

## What is in it?

- [certbot_hetzner](roles/certbot_hetzner/): uses Hetzner's DNS system and certbot to generate *Let's encrypt* certificates
- [openvpn_client](roles/openvpn_client/): installs an *OpenVPN* client on the system that can optionally also act as a gateway - based on client configs generated by the openvpn_server-role.
- [openvpn_server](roles/openvpn_server/): installs an *OpenVPN* server on the system, configures it and generates client configs
- [ssh_users](roles/ssh_users/): installs linux users, enables ssh access and hardens ssh.
- [update_debian_ubuntu](roles/update_debian_ubuntu/): updating of the system


## Conventions used in this collection

Local:
- `artifacts`: stuff downloaded from hosts goes into `"{{ inventory_dir }}/artifacts/{{ host }}`

Remote:
- Software: all custom software such as scripts, checkouts, docker-compose should live inside of `/opt/<app>` on the host.
- Data: all data of services that is to be persisted should live in `/srv/<app>/` on the host.
